Asset Creation APP - User Guide

Introduction

Welcome to the user guide of the Asset Creation APP! This guide will help you to create and upload assets to our server. With this guide, you will be able to create assets that meet the requirements and use some extra functionalities

Step 1: Navigate to the Asset Creation Webpage

In this guide, we will use Google Chrome as an example. To access the Asset Creation Webpage, open the bootStrap.html link in your web browser. Then, please ensure that your screen size is between 992 and 1200 pixels to properly view the Asset Creation APP Webpage. The Asset Creation APP has the following components:

  • User Name
  • Help Menu
  • Side Bar Switch
  • List of Assets in Best Condition Menu
  • Daily Reporting Rates Graph â€“ All Users Menu
  • Base Map
  • Your Asset Points
Step 2: View the Existing Assets

The existing assets are represented as markers on the map. When you click on a marker, a pop-up message will appear displaying the name and latest condition description of the asset. The markers are color-coded from green to red according to their latest condition:

Asset is in very good condition

Some aesthetic defects, needs minor repair

Functional degradation of some parts, needs maintenance

Not working and maintenance must be done as soon as reasonably possible

Not working and needs immediate, urgent maintenance

No condition captured

Step 3: Fill in your Asset Information

To create a new asset, click anywhere on the map. A form will pop up, displaying the latitude and longitude of the location you clicked, you will be asked to fill in the following information:

  • Asset Name: Enter a unique name for your asset.

    Please ensure that the asset name you enter is unique, as our APP will not insert the asset if the name is already in use and will instead pop an alert message.

  • Installation date: Select an installation date for your asset.

Step 4: Upload your Asset

Once you have filled in the Asset Information, you can upload your asset by clicking on the "saveAsset" button.

After you have successfully created the asset, you will receive an alert message confirming its creation. Then You can view the newly created asset marker on the map.

Additional Functionality 1: List of Assets in Best Condition

You can view a list of all assets that have at least one report (at any point in time) indicating they are in the best condition by clicking on the "List of Assets in Best Condition" menu in the sidebar.

You can view a graph that shows daily reporting rates for the past week, including the number of reports submitted and how many of them had a condition listed as one of the two "not working" options for all users. This can be done by clicking on the "Daily Reporting Rates Graph â€“ All Users" option located in the sidebar.

The x-axis represents the day of the past week, while the y-axis represents the count of reports.

Why do I get these alert messages?

When running the APP, you may have these error alert messages. Here are some prompts for them:

  • "Failed to load the condition details.": This error message pops when the APP is unable to retrieve the condition details from the server. To resolve this issue, check your internet connection and try reloading the page. If the issue persists, please contact the administrator for assistance.

  • "Failed to get the user id.": This error message pops when the application is unable to retrieve the user's ID. To resolve this issue, check your internet connection and try reloading the page. If the issue persists, please contact the administrator for assistance.

  • "Failed to load the created assets of user.": This error message pops when the application is unable to retrieve the assets created by the user. To resolve this issue, check your internet connection and try reloading the page. If the issue persists, please contact the administrator for assistance.

  • "Please insert asset name!": This error message pops when the asset name input field is left blank when creating a new asset. To resolve this issue, insert an asset name and try again.

  • "Please insert installation date!": This error message pops when the installation date field is left blank when creating a new asset. To resolve this issue, select an installation date from the calendar and try again.

  • "The asset name is not unique. Please choose a different one!": This error message pops when the asset name you entered already exists. To resolve this issue, choose a different and unique asset name and try again.

  • "Failed to insert the asset.": This error message pops when the application is unable to insert the asset into the database. To resolve this issue, check your internet connection and try again. If the issue persists, please contact the administrator for assistance.
